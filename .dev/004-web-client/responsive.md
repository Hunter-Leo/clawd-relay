# 004 — Web Client 响应式布局规范

> Aligned with design.md DIALS: DESIGN_VARIANCE=3 · MOTION_INTENSITY=4 · VISUAL_DENSITY=6

## 1. 断点系统

使用 Tailwind 标准断点：

| 断点 | 最小宽度 | 设备 | 策略 |
|------|---------|------|------|
| `sm` | 640px | 大屏手机横屏 | 微调间距 |
| `md` | 768px | 平板竖屏 | 两栏布局出现 |
| `lg` | 1024px | 平板横屏/小屏桌面 | 三栏布局 |
| `xl` | 1280px | 标准桌面 | 内容区域定宽 max-w-7xl |
| `2xl` | 1536px | 大屏桌面 | 与 xl 一致，间距放大 |

默认（`<sm`，即 <640px）为手机竖屏布局。

## 3. 响应式行为规范

### 3.1 设备分组列表

| 属性 | Desktop (≥lg) | Tablet (md) | Mobile (<md) |
|------|--------------|-------------|--------------|
| 网格列数 | `grid-cols-3` | `grid-cols-2` | 堆叠（无网格） |
| 设备分组间距 | `gap-4` | `gap-3` | `space-y-3` |
| 设备名行 | 展开可见 | 展开可见 | 折叠 + 展开箭头 |
| 卡片默认显示 | 全部展开 | 全部展开 | 折叠（仅设备名+状态+计数） |

### 3.2 SessionCard

| 属性 | Desktop (≥lg) | Tablet (md) | Mobile (<md) |
|------|--------------|-------------|--------------|
| 内部 padding | `p-4` | `p-3` | `p-3` |
| 标题行 | agent + title + state 同行 | 同桌面 | 标题换行 |
| toolInput | 显示摘要（1 行截断） | 显示摘要（1 行截断） | 仅显示 "toolName" |
| CWD 路径 | 完整显示（截断过长） | 截断保留末尾 | 隐藏 |
| 子会话计数 | 显示 badge | 显示 badge | 仅显示数字 |

### 3.3 PermissionModal

| 属性 | Desktop (≥lg) | Tablet (md) | Mobile (<md) |
|------|--------------|-------------|--------------|
| 宽度 | `max-w-[560px]` | `max-w-[90vw]` | `max-w-[calc(100vw-32px)]` + `mx-4` |
| 位置 | 居中 | 居中 | 距离底部 48px（更易单手操作） |
| 按钮布局 | 水平排列 3 按钮 | 水平排列 | 垂直堆叠（全宽按钮） |
| toolInput 区域 | `max-h-48` | `max-h-36` | `max-h-24` |
| 关闭按钮 | 右上角 × | 右上角 × | 右上角 × + 底部取消 |

### 3.4 SessionHUD

| 属性 | Desktop (≥lg) | Tablet (md) | Mobile (<md) |
|------|--------------|-------------|--------------|
| 位置 | 右下角浮动 | 右下角浮动 | 底部固定 bar |
| 样式 | 紧凑垂直列表 | 紧凑垂直列表 | 水平滚动列表 |
| 内容 | 设备名+状态缩写 | 设备名+状态缩写 | 仅状态圆点+设备缩写 |
| 可见性 | 始终可见 | 始终可见 | 滑动隐藏（观看模式） |

### 3.5 连接指示器

| 属性 | Desktop (≥lg) | Tablet (md) | Mobile (<md) |
|------|--------------|-------------|--------------|
| 位置 | 页面顶部栏右侧 | 顶部栏右侧 | 顶部栏右侧 |
| 显示 | 圆点 + 文字 | 圆点 + 文字 | 仅圆点 |
| 详细状态 | hover tooltip | 点击浮窗 | 点击弹出 bottom sheet |

### 3.6 设置面板

| 属性 | Desktop (≥lg) | Tablet (md) | Mobile (<md) |
|------|--------------|-------------|--------------|
| 展示方式 | 右侧滑出面板 / 模态 | 右侧滑出面板 | 全屏页面 / bottom sheet |
| 宽度 | `w-[400px]` | `w-[360px]` | 全宽 |
| 分组布局 | 双列 | 单列 | 单列 |

## 4. 导航适配

桌面端不存在传统导航栏 —— 页面本身是 Dashboard。在移动端：

- 顶部栏：左侧 `ConnectionIndicator`（仅圆点），右侧 `DNDToggle` + `SettingsButton`
- 无汉堡菜单（单页面无多路由）
- 如果后续需要多页面视图，在移动端使用底部 tab bar（3-4 个 tab）

## 5. 触摸交互

### 移动端手势

| 手势 | 操作 |
|------|------|
| 左滑 SessionCard | 关闭/忽略该会话（若支持） |
| 下划 PermissionModal | 关闭模态框 |
| 长按设备名 | 展开/折叠设备分组 |
| 点击状态指示器 | 显示详细连接信息（popover） |

### 触摸目标尺寸

- 所有可点击元素最小高度 44px（iOS 准则）
- 按钮最小宽度 48px
- 开关/复选框最小尺寸 28px

## 6. 安全区域

移动端（特别是 iOS Safari）需要处理安全区域：

```css
/* 顶部 safe area */
.inset-safe-top {
  padding-top: env(safe-area-inset-top, 0px);
}
/* 底部 safe area（SessionHUD） */
.inset-safe-bottom {
  padding-bottom: env(safe-area-inset-bottom, 0px);
}
```

- `SessionHUD` 在移动端固定在底部时添加 `padding-bottom: env(safe-area-inset-bottom)`
- 页面内容不进入安全区域

## 7. 不可使用的布局

- 不在桌面端使用汉堡菜单（页面无多路由）
- 不使用 `h-screen`（改用 `min-h-[100dvh]`，避免 iOS Safari 地址栏跳动）
- 不使用 `position: fixed` 将大元素固定在桌面端（仅 ConnectionIndicator 和 SessionHUD 使用）
- 不使用 `calc()` 做复杂百分比布局（全部使用 CSS Grid）

## 8. 布局一致性检查清单

开发阶段逐项检查：

- [ ] 每个多列布局声明了 `<768px` 的 fallback
- [ ] 所有骨架屏形状匹配最终内容
- [ ] 移动端点按目标 ≥ 44px
- [ ] iOS Safari safe area 正确处理
- [ ] `min-h-[100dvh]` 替代 `h-screen`
- [ ] 权限气泡在移动端垂直堆叠按钮
- [ ] 无 `hamburger` 菜单（除非后续多路由）
- [ ] 英文长文案不导致按钮换行
- [ ] 暗色/亮色模式在移动端验证
